### v2.0.0 - Compliance Platform Overhaul

<Update label="2024-09-15" description="v2.0.0" tags={["feature", "breaking"]}>

## New Features

- Launched unified compliance dashboard integrating risk scores, sanctions screening, and transaction graphs.
- Integrated support for 20+ new blockchains including Solana and Polygon.

## Breaking Changes

- Deprecated legacy `/v1/risks` endpoint; migrate to `/v2/risks`.
- Updated authentication to require JWT tokens instead of API keys for enhanced security.

<CodeGroup tabs="JavaScript,Python">
```javascript
// New v2 endpoint usage
const risks = await fetch('https://api.merklescience.com/v2/risks', {
  headers: { 'Authorization': `Bearer ${YOUR_JWT_TOKEN}` }
});
```
```python
# New v2 endpoint usage
import requests
response = requests.get(
    'https://api.merklescience.com/v2/risks',
    headers={'Authorization': f'Bearer {YOUR_JWT_TOKEN}'}
)
```
</CodeGroup>

</Update>

## Migration Guide

Follow these steps to upgrade from v1.x to v2.x and handle deprecations.

<Steps>
  <Step title="Update Authentication" icon="shield">
    Generate JWT tokens via the new auth endpoint.

    ```bash
    curl -X POST https://api.merklescience.com/auth/token \
      -d "client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET"
    ```
  </Step>
  <Step title="Replace Deprecated Endpoints" icon="code">
    Update all calls from `/v1/risks` to `/v2/risks`.
  </Step>
  <Step title="Test New Features" icon="play">
    Validate behavioral rules with sample illicit flows.
  </Step>
</Steps>
